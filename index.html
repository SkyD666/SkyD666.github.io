RecyclerView

首先，Recyclerview是Android 5.0推出的

什么是RecyclerView？

就是每一项都是一个子Item，然后从上到下排列显示（其实学到后面你会发现也可以横向排列，网格状排列等，还有一些更高级的显示方式……这里就不再多讲）

接着讲一下适配器模式

适配器模式

首先，举个例子，读卡器是作为内存卡和笔记本之间的适配器。把内存卡插入读卡器，再将读卡器插入笔记本，这样就可以通过笔记本来读取内存卡。

然后再来看定义，适配器模式（Adapter Pattern）是作为两个不兼容的接口之间的桥梁。把一个类的接口变换成客户端所期待的另一种接口，从而使原本因接口不匹配而无法在一起工作的两个类能够在一起工作。

然后我们在Activity类中通过findViewById来找到它。

然后我们在新建一个类，叫做RvData，用来存放每一项的数据。由于我们的布局里面只有一个TextView，所以这里我们只存放字符串来让TextView显示就好了。

 

2.4

想要让RecyclerView工作起来，我们还需要有一个Adapter适配器。

这里我们新建一个类，叫做RvAdapter，然后继承RecyclerView.Adapter并传入泛型（需要传入一个RecyclerView.ViewHolder或其子类的类型），这里我们就传入RvViewHolder（这个东西是对应每一项布局的一个类，里面存放我们xml写的那个TextView变量），然后重写onCreateViewHolder、onBindViewHolder、getItemCount三个方法，同时，我们需要在构造RvAdapter时传入每一项的数据列表List<RvData> list。

然后onCreateViewHolder就是创建每一项的ViewHolder，它的返回值就是上图的RvViewHolder。至于LayoutInflater，可以先认为是把上面写的xml文件加载了进来。

onBindViewHolder就是对每一项进行赋值。我们对刚才的TextView的文字进行设置，内容就是当前RvData对象里的name字符串

getItemCount就是取得所有项目的个数，所以我们直接返回list.size()即可

 

2.5

然后我们再在Activity中对recyclerView进行设置。

我们先增加一个list变量，并用ArrayList初始化（因为List只是一个接口，里面有方法并没有实现，所以我们可以用已经实现了接口的ArrayList来初始化），然后我们增加一个我们刚才RvAdapter类型的适配器rvAdapter变量，并将list传入。

然后，因为我们只需要从上到下线性的显示一个列表就可以了，所以我们新建一个线性布局管理器(LinearLayoutManager)这里是用的一个参数的构造函数（还有GridLayoutManager表格布局管理器和StaggeredGridLayoutManager瀑布流布局管理器（瀑布流就是快手主页那些交错显示的样式，也可见下图），用来实现更多的显示样式，同学们可以下课学习一下），并设置布局管理器setLayoutManager。

当然你在初始化LinearLayoutManager时，也可以用三个参数的构造函数，第一个参数是context上下文，第二个参数是滚动方向，可以有两种LinearLayoutManager.VERTICAL竖直滚动和LinearLayoutManager.HORIZONTAL水平滚动，第三个参数是滚动方向是否反转，如果是false，则就是从上向下滚动、从左到右滚动；如果是true，就是从下往上滚动，从右向左滚动。

然后我们对list进行添加一些数据。添加完成后要及时调用adapter的notifyDataSetChanged方法，来告诉adapter，list里面的数据更新了。以便RecyclerView及时更新界面。

其实除了notifyDataSetChanged，还有许多其它类似的方法，有删除某一项后调用的，有插入某一项后调用的，还有将某一项移动位置后调用的……

2.6 这样RecyclerView的一个简单的使用就完成了

Fragment

Fragment，简称碎片，是Android 3.0（API 11）提出的。其依赖于Activity，但是拥有自己的活动事件与生命周期。一个Activity里可以有多个Fragment。

QQ里的Fragment，底部三个导航栏按钮，对应三个Fragment，整个大页面是一个Activity。

1.使用

在包上右击，New->Fragment->Gallery，然后选择Fragment (Blank)即可。

自动创建好的Fragment 有很多方法和变量，目前我们只要onCreate和onCreateView就好了，其他的可以删掉。

Fragment生命周期

onCreate

系统会在创建时调用此方法，此生命周期只会在整个Fragment生命周期里只会调用一次。

onCreateView

创建Fragment的绘制视图界面，此生命周期只会在整个Fragment生命周期里只会调用一次。

此生命周期也是通常是View的控件findViewById的地方

onStart

此生命周期用户已经可见Fragment，但没有焦点，与用户不能交互。此生命周期可以被多次调用。

onResume

对用户可见并正在运行时调用，并且用户可以交互操作页面。此生命周期可以被多次调用。可以重新恢复视频播放。

onStart()和onResume()的区别

onStart()是界面被显示出来的时候执行的，用户可见，但没有将它完全覆盖，不能与它交互 onResume()是能与用户能进行交互时被执行，用户可以获得焦点，能够与用户交互。

onPause

Fragment进入后台，用户即将不可见时调用。可以在这里暂停视频播放。

onStop

Fragment已经进入后台时调用，这个时候一瞬间的页面无法操作。这个生命周期下操作View会报错。

onDestroyView

视图已经与Fragment分离时调用，如果要恢复这个Fragment的视图，需要重新进入onCreateView。

onDestroy

当Fragment不再使用时调用，可以在这里释放资源。

onDetach

Fragment与Activity分离时调用。

动态添加/替换Fragment

1 创建待添加的碎片实例 2 获取FragmentManager，在Activity中直接通过调用getSupportFragmentManager()方法得到 3 通过beginTransaction()方法开启一个事务 4 向容器内添加/替换碎片，使用add()/replace()方法实现 5 使用commit()方法来提交事务

另外可以与ViewPager结合使用，让每个ViewPager页面是一个Fragment。

一般来说大多数时候TabLayout是与ViewPager和Fragment一起使用。

首先在AS的Android结构下，在Gradle Scripts下的build.grade (Module: app)文件里添加

implementation 'com.google.android.material:material:1.2.1'

然后在AS点击右上角的Sync Now。

2. xml

然后我们在对应的Activity的xml文件里添加TabLayout和ViewPager

tabIndicatorColor 就是下图中绿色下划线的颜色

tabIndicatorHeight 就是下图中绿色下划线的高度

tabSelectedTextColor 就是下图选中后文字的颜色

 

若 tabIndicatorGravity 属性设置为stretch，则上面的绿色下划线会占满整个高度

 

若设置为center，则会在中间显示。

 

若设置为bottom，会在底部显示，也就是默认的样式。

 

若设置为top，会在顶部显示。

 

tabMode有两种属性：fixed和scrollable；fixed：指的是固定tab；scrollable指的是tab可滑动。

tabGravity有两种属性：center和fill；center指的是居中显示，fill指的是占满全屏。

3. Activity里代码编写

首先我们要有一个ViewPager的Adapter，如下。fragmentList是存放Fragment的List，fragmentTitle是存放标题的List

其中getPageTitle方法就是获取ViewPager每一页的标题，在这里的作用就是TabLayout上面显示的文字。

x
class ViewPagerAdapter extends FragmentPagerAdapter {
    ViewPagerAdapter(FragmentManager fm, int behavior) {
        super(fm, behavior);
    }
​
    @Override
    public int getCount() {
        return fragmentList.size();
    }
​
    @NonNull
    @Override
    public Fragment getItem(int position) {
        return fragmentList.get(position);
    }
​
    @Nullable
    @Override
    public CharSequence getPageTitle(int position) {
        return fragmentTitle.get(position);
    }
}

然后我们在设置好fragmentList和fragmentTitle后，需要设置ViewPager的Adapter，并且调用TabLayout的setupWithViewPager方法来让ViewPager和TabLayout协同工作（改变任意一个的页面，另一个也会跟着改变）。

ViewPagerAdapter()的第二个参数有两个值：

{BEHAVIOR_SET_USER_VISIBLE_HINT, BEHAVIOR_RESUME_ONLY_CURRENT_FRAGMENT}

第一个值：只有在滑动这个Fragment上时才会调这个Fragment的onResume()方法。但是注意是回调onResume()。而onResume之前的方法，已经在getItem()中实例化Fragment的时候调完了。

第二个值：只会执行到onStart()。

第一个值已不被官方推荐使用，原因：就是ViewPager+Fragment"诡异"的缓存特性。无论Fragment可见或者不可见，只要是存在于ViewPager这个容器之内的Fragment，都会统一执行生命周期函数，如果ViewPager有10个fragment，但是一次只显示了一个，你离开了当前Activity之后回来，Activity执行 onResume，然后10个Fragment 一起执行 onStart-onResume , 而恰好你在onResume里面写了耗时操作 , 关键是其他9个不可见，系统资源完全浪费。因此用第二个值只执行到onStart()即可。

（可以演示，可能会遇到AS 4.1更改代码后运行不生效的bug，需要clean project一下）

x
viewPager.setAdapter(new ViewPagerAdapter(getSupportFragmentManager(),
                ViewPagerAdapter.BEHAVIOR_RESUME_ONLY_CURRENT_FRAGMENT));
tabLayout.setupWithViewPager(viewPager);

Material Design

由于之前很多应用界面风格不统一，后来谷歌想让所有应用就有统一的外观，就像是苹果向开发者提出的设计原则一样。谷歌还基于这种新的设计语言对本公司旗舰应用进行了重新设计，包括安卓和网页端的Gmail和Calendar。经过了重新设计的Gmail，界面更干净、更简约。在安卓平台上，这种新界面被称为Material，支持各种新动画效果，具有内置的实时UI阴影……

首先，我们来欣赏17个使用Material Design风格的APP界面

https://www.shejidaren.com/17-apps-using-material-design.html

常用控件

1.Toolbar

Toolbar 是在 Android 5.0 开始推出的一个 Material Design 风格的导航控件 ，Google 非常推荐大家使用 Toolbar 来作为Android客户端的导航栏，以此来取代之前的 Actionbar 。与 Actionbar 相比，Toolbar 明显要灵活的多。它不像 Actionbar 一样，一定要固定在Activity的顶部，而是可以放到界面的任意位置。

2.FloatingActionButton

FloatingActionButton是一个继承ImageView悬浮的动作按钮，经常用在一些比较常用的操作中，一个页面尽量只有一个FloatingActionButton，否则会给用户一种错乱的感觉！

3.Snackbar

Snackbar就是一个类似Toast的快速弹出消息提示的控件，手机上显示在底部，大屏幕设备显示在左侧。但是在显示上比Toast丰富，也提供了于用户交互的接口

4.BottomNavigationView

BottomAppBar是底部导航栏。

5.DrawerLayout
